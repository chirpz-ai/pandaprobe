The PandaProbe dashboard provides a visual interface for creating eval runs, selecting metrics, configuring filters, and reviewing results. This page will document the full dashboard workflow.

## Planned features

### Creating a trace eval run

1. Navigate to the **Evaluations** section in the dashboard
2. Select one or more trace-level metrics (e.g., `task_completion`, `tool_correctness`)
3. Configure filters to target specific traces (date range, status, session, tags)
4. Set a sampling rate to control how many traces are evaluated
5. Optionally override the LLM model used for judging
6. Submit the eval run and monitor progress

### Creating a session eval run

1. Select session-level metrics (`agent_reliability`, `agent_consistency`)
2. Configure session filters (date range, user, minimum trace count)
3. Optionally customize signal weights
4. Submit and monitor the session eval run

### Reviewing results

- View per-trace and per-session scores with reasons and metadata
- Track score trends over time with built-in charts
- Explore score distributions across metrics
- Compare session scores in a leaderboard view
- Drill into flagged traces from agent reliability results

### Manual annotations

- Score individual traces manually from the trace detail view
- Edit existing automated scores to correct or override them
- All annotations are tracked as `ANNOTATION` source scores
