## Prerequisites

Before running evaluations, make sure you have:

<Steps>
  <Step title="Traces or sessions to evaluate">
    Evaluations run against data already captured by PandaProbe tracing. You need at least one trace in your project. For session-level evaluations, you need traces grouped by session ID.
  </Step>
  <Step title="An LLM provider configured">
    Most trace-level metrics use an LLM-as-judge approach. PandaProbe needs access to at least one LLM provider (e.g., OpenAI, Anthropic) to run these metrics. Check available providers:

    ```bash
    curl -H "Authorization: Bearer $TOKEN" \
         -H "X-Project-ID: $PROJECT_ID" \
         https://api.pandaprobe.com/evaluations/providers
    ```
  </Step>
  <Step title="Authentication">
    All evaluation endpoints require authentication. You can use either:
    - **Bearer JWT** with `X-Project-ID` header
    - **API key** with `X-API-Key` and `X-Project-Name` headers
  </Step>
</Steps>

## What you can evaluate

### Trace-level evaluation

Run metrics against individual traces to score task completion, tool usage, planning, coherence, and more. You can target traces by:

- **Filters** — date range, status, session, user, tags, or name substring
- **Explicit IDs** — provide a list of specific trace UUIDs
- **Sampling** — evaluate a random fraction of matching traces to control cost

### Session-level evaluation

Run metrics against entire sessions to assess agent reliability and consistency. You can target sessions by:

- **Filters** — date range, user, error status, tags, minimum trace count
- **Explicit IDs** — provide a list of specific session ID strings
- **Sampling** — evaluate a fraction of matching sessions

## Understanding eval run results

Every eval run produces **scores** attached to your traces or sessions. Each score includes:

| Field | Description |
|---|---|
| `name` | The metric name (e.g., `task_completion`) |
| `value` | Score as a string (e.g., `"0.85"`) |
| `status` | `SUCCESS` or `FAILED` |
| `reason` | LLM-generated explanation |
| `metadata` | Structured data with threshold, intermediate results, and breakdowns |

You can query scores in aggregate (summaries, trends, distributions) or per trace/session to understand individual results.

## Manual scores

Beyond automated evaluations, you can create scores manually:

- **Annotations** — human reviewers score traces in the dashboard
- **Programmatic** — submit scores from your own evaluation pipelines via the API

Manual scores use the same schema as automated scores and appear alongside them in analytics.
